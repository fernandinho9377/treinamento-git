Perguntas e Respostas


1) Quais os comandos para configurar usuário e email no git?
Para se configurar usuário e email utilizamos os comandos:
git config --global user.name "w3schools-test"
git config --global user.email "test@w3schools.com"



2) Qual a diferença entre git e github?
Git é um sistema de controle de versão popular. Foi criado por Linus Torvalds em 2005, e desde então é mantido por Junio ​​Hamano.
É usado para:
-Alterações no código de acompanhamento
-Rastreamento de quem fez alterações
-Colaboração de codificação
Git não é o mesmo que GitHub. O GitHub cria ferramentas que usam Git. O GitHub é o maior host de código-fonte do mundo e pertence à Microsoft desde 2018.

3) O que faz o comando commit?
Uma vez que terminamos nosso trabalho, estamos prontos para mudar stagepara commit nosso repo.
A adição de commits acompanha nosso progresso e mudanças enquanto trabalhamos. O Git considera cada commitponto de mudança ou "save point". É um ponto no projeto ao qual você pode voltar se encontrar um bug ou quiser fazer uma alteração.
Quando nós commit, devemos sempre incluir uma mensagem .
Ao adicionar mensagens claras a cada commit, é fácil para você (e para os outros) ver o que mudou e quando.

4) O que faz o comando push? O comando push transfere commits do repositório local a um repositório remoto. É o oposto do comando git fetch , que importa commits para branches locais, enquanto o comando push exporta commits para branches remotos.

5) O que faz o comando pull? O comando git pull é usado para buscar e baixar conteúdo de repositórios remotos e fazer a atualização imediata ao repositório local para que os conteúdos sejam iguais. Fazer o merge de alterações upstream remotas no repositório local é algo comum em fluxos de trabalho de colaboração baseados em Git.
